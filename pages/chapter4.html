"A function is a name given to a series of statements. When you call the function, it runs those statements. Functions take 0 or more arguments, do something with them, and then return something. type(variableName) from earlier was an example of a function. The function itself was called type(foo), the argument was variableName, and the return value was the type of variableName.

"Python already has many functions defined. I will talk about just a few of them. There is print(foo), which will take what you put in and print it out to the screen. You can put anything you like in there, whether it be some constants or a variable:

>>>print(25)
25
>>>print(12.0)
12.0
>>>print("ebola")
ebola
>>>words = "Never gonna run around and desert you"
>>>print(words)
Never gonna run around and desert you
>>>x = 3
>>>print(x)
3

"There's also a function to accept input from the keyboard, input(). When this is called, whatever argument you put will be displayed, and the cursor will stay where it is and flash, waiting for the user to type something in and hit Enter. This lets the user set values in the computer, rather than having you write all values before running. The user's input below is underlined to show the difference. input() returns whatever it reads in.

>>>input("What's your name? ")
What's your name? Kunta
'Kunta'

"There are functions to convert variables into other types, which are just the names of those types. int(foo), float(foo), and str(foo) respectively try to turn whatever is given to them into themselves. Remember that floats can be converted into ints, but everything after the decimal point will be lost. If the function doesn't succeed at converting its argument, then it'll give you an error message.

"You can call functions in the arguments of other functions. This is called function composition. It will work from the inside out.

>>>x = '6' # Right now, x is a string
>>>print(int(x) + 4)
10

"And finally, you can create your own custom functions using the def keyword. The syntax for writing your own functions looks like this:

>>>def functionName(foo, ... , bar): ## This is the header.
    The body of the function; where the magic happens
    return foobar

"Just like before, you have to indent everything after the function name, which shows that it's inside the scope of that function. The rules for function names are the same as the rules for variable names. You don't have to return anything; if you don't, then you end the function with an empty line. From then on, you can call this function by entering functionName() with the appropriate variables, and it will do everything in the body of the function.

"As a final side note, functions are more useful in saved script .py files. You don't have to run things through the interpreter all the time. Instead, you can create a new file with the shortcut Ctrl+N, and then save it like you would save a document in Microsoft Word, Notepad, or a picture in Paint: Ctrl+S and enter a filename. All the commands you write in the script file will be carried out when you run the script. This isn't important for this tutorial, but you will probably need it someday, so I'll mention it now so that it doesn't blindside you later.

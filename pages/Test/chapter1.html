Your eyes follow his pointing finger, and you spot some weird blue things floating in front of you. They look like clouds of blue squares, fluttering around and changing size nearly faster than your eyes can keep up with. What the heckity?

As you watch the blue things undulate gently, the red guy helpfully explains, "What you see here is data. These chunks are called variables, and each variable has a value. Variables are one of the main things that the computer actually works on. In Python, there are a few different types of variables that are commonly used, like ints, strings, and floats.

"Int: That's short for "integer." You might know from your math classes already, but these are whole numbers. They can be positive or negative. Some examples of ints are 12, 0, 42, and -10000.

"String: These are like combinations of words and letters. You can have any character you want be part of a string. Some examples of strings are hi, hello world, a, and abc123. Be careful! There's a difference between the int 1 and the string "1". Strings are written with quotes around them to help differentiate between cases like these.

"Float: That's short for "floating point number." These are basically any number that has a decimal point in it, like fractions. Some examples of floats are 1.5, 0.33333, 50.0, and -12.9. Doing math with floats is not always 100% accurate because of how the computer stores information, but you shouldn't need to worry about that yet. In general, ints are more precise. Just like how there are strings that look like ints, there are also floats that look like ints. 1.0 is different from 1! If you want to assign some variable to be a float, you should include the decimal point.

"Bool: These only have two values, True and False. They have their uses, which you might be able to come up with later.

"To declare a variable, you just say its name, and assign it a value with the assignment operator, =. Variable names are case sensitive! So x and X are different variables to the system! These names can't have any spaces in them. By convention, we give them names that start with lowercase letters, and if they're more than one word long, all the words after the first have their first letters capitalized. This is called camelcase, and it helps to read the name because you can tell where one word starts and the next begins. Separating different words with underscores (_) is also accepted, but I will use camelcase for the rest of this. For example: variableName is camel-case. VariableName, variablename and vaRIaBlENaME would also be accepted, but they're not great. Try to avoid starting variable names with uppercase letters or numbers, and definitely avoid using special characters like % and ! in them!

variableName = 5

"You can find out what the type of a variable is by using this command: type(variableName).

>>>type(variableName)

<class 'int'>

"You can even set variables equal to each other! Keep in mind that whatever's on the right side of the = has to have a value already.

>>>x = 1

>>>y = x

>>>y
